◆ Call()

rev::Call::Call ( std::function< void(void)> icallback)

Construct a new Call segment.

Parameters
icallbackThe function which will be invoked for this segment

This function should be non-blocking. This means it should perform some action and return ASAP, without any artificial delays or control loops. If you want to use this to, say, start moving an arm controlled by a PID toward a target, you should instead have a separate thread for that, and use the Call segment to set a variable that tells that thread to begin execution.

Calculate the next step.

This method is executed once per step of the controller, until the segment is completed, beginning with the same cycle as init(), immediately after.

Upon reaching a return value of NEXT, the controller is intended to step to the next segment, or if no next segment is present, terminate execution.
